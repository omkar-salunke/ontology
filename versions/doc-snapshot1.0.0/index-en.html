6. Change log back to ToC

The following changes have been added since the previous draft:

  • Added property hasFileName (functional)
  • Added property hasID (functional)
  • Added property hasLocation (dataset->Location)
  • Renamed datasetCreator to author.
  • Added domain and range for author and contributor.
  • Added domain for hasDOI (Software union dataset union publication)
  • Added data property title.
  • Deleted property mbox(foaf)
  • Added class Person (quivalent to foaf:Person)
  • Added data property email (Person -> string)
  • Added data property citation (Publication -> Literal) subclass of dc:bibliographicCitation.
  • Added data property jouran (Publication -> string)
  • Added range to data property hasUnits
  • Removed float from hasValue to avoid overloading the term.
  • Renamed datasetYear to datasetDate
  • Added range int to publicationYear.
  • Added property hasISBN (Publication -> string)
  • Added property hasISSN (Publication -> string)
  • Added property includesPaleoData (Dataset -> PaleoData)
  • Added property includesChronData (Dataset -> ChronData)
  • Added external properties for lat, long and alt (of a point in a location)
  • Added geosparql properties for referring to Locations and their geometries.
  • Added labels to all the terms.
  • Added definitions to all the terms.
  • Added 4 subclasses of DataTable.
    • MeasurementTable: contains the measuredVariables made on the Archive
    • EnsembleTable is a table containing possible series of age models or proxy variability
    • SummaryTable: relevant statistics from the ensemble.
    • DistributionTable contains the probability on the inferred variables.
  • Added the following properties to refer to the respective tables (Data or Variable -> DataTable):
    • foundInMeasurementTable
    • foundInModelTable
    • foundInEnsembleTable
    • foundInInterpretationTable
  • Added calibratedFrom (MeasuredVariable->CalibrationModel)
  • Add data property hasExecutionEnvironment (SoftwareCode->String)
  • Added data property hasExecutionCommand (SoftwareCode->String)
  • Changed the domain of modeledBy adding Data. The range is Model
  • chronModeledBy added as a subPropertyOf modeledBy (domain ChronData (instead of Data) -> range PaleoModel)
  • paleoModeledBy added as a subPropertyOf modeledBy (domain PaleoData (instead of Data) -> range ChronModel)
  • PaleoModel added as a subClassOf Model
  • ChronModel added as a subClassOf Model
  • foundInSummaryTable has now domain [paleoModel or chronModel]
  • foundInEnsembleTable has now domain [paleoModel or chronModel]
  • foundInDistributionTable has now domain [paleoModel or chronModel]
  • [hasCode, foundInSummaryTable, foundInEnsembleTable are functional properties now
